üî¢ Version Control Rules

Rule 1: Increment on Save
Every time you make significant changes, save as new version: v1 ‚Üí v2 ‚Üí v3. Never overwrite.
Rule 2: Archive Old Versions
Move previous versions to /Archive/ subfolder. Highest version in main folder = current.
Rule 3: No Offline Copies
Always work from the shared drive. If you need offline access, use folder sync (OneDrive/Google Drive sync) ‚Äî don't download separate copies.
Rule 4: Mark in Trello
When you upload a new version, comment in Trello: "Uploaded v2 of [Section] - [brief note of changes]"

üö´ Common Mistakes to Avoid

  • DON'T: Download files to your desktop and edit offline
  • DON'T: Name files with dates like "SIGMA_Geology_MB_15Jan.docx"
  • DON'T: Save files outside the project folder structure
  • DON'T: Overwrite existing files without incrementing version
  • DON'T: Send files via email/WhatsApp instead of using shared drive
  • DO: Always check you're editing the highest version number
